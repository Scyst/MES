📋 System Modules Update (เจาะลึกรายละเอียดงาน)

1. OEE Dashboard & Shopfloor Analytics

Major Update
🎯 ปัญหา & เป้าหมายที่ต้องการแก้ไข

ผู้บริหารและหัวหน้างานต้องการวิเคราะห์ข้อมูล OEE, ประสิทธิภาพ และของเสีย โดยแบ่งตาม "ทีมผลิต (Team)" เพื่อเปรียบเทียบผลงานแต่ละกะ แต่ระบบเดิมไม่รองรับการกรองข้อมูลระดับทีมบนหน้า OEE Dashboard นอกจากนี้ยังมีปัญหาภาพกราฟิก UI เพี้ยนเมื่อสลับตัวกรองข้อมูล

✨ สิ่งที่พัฒนาสำเร็จในสัปดาห์นี้

  • จัดทำเอกสารคู่มือ Architecture & Database Structure ของระบบ OEE Dashboard ทั้งหมด เพื่อเป็นมาตรฐานในการพัฒนาต่อยอด
  • อัปเกรด Stored Procedures 6 ตัว และปรับปรุง API ให้รองรับพารามิเตอร์ @Team สำหรับวิเคราะห์ยอดผลิตและต้นทุนตามกะการทำงาน
  • เพิ่มกล่องเลือก Team Filter ลงในแถบเครื่องมือของหน้า OEE Dashboard & Shopfloor โดยดึงรายชื่อทีมจากฐานข้อมูลพนักงานแบบอัตโนมัติ
  • แก้ไขบัคเชิงลึกของ HTML5 Canvas (Chart.js) ที่ทำให้กราฟวงกลมวาดทับซ้อนกันจนเกิดเส้นสีประหลาดเวลาเปลี่ยนตัวกรอง (Context State Corruption Fix)
  • แก้ไขปัญหา CSS Border Rendering บนหน้าจอการ์ดสรุปยอดเงิน (Revenue/Good/Scrap) ที่เส้นขอบ 1px ตีกันจนภาพแหว่ง โดยปรับให้ไฮไลต์เฉพาะแถบสีหนาด้านซ้ายเท่านั้น
Business Impact: ผู้บริหารสามารถติดตามประสิทธิภาพการผลิตและประเมินผลงาน (KPI) ของแต่ละทีม/กะ ได้อย่างแม่นยำ และระบบ UI ทำงานได้อย่างเสถียรไม่มีอาการกระตุกหรือกราฟิกแตกเมื่อดึงข้อมูลใหม่
