# ปรับปรุงระบบ Projects ตาม Feedback

จากที่คุณเสนอมา เป็นแนวคิดที่ยอดเยี่ยมมากครับ การใช้ Checklist เป็นแกนกลางในการวัด Progress จะตอบโจทย์การทำงานจริงมากกว่าการนับจำนวน Tasks ที่ถูกสร้างขึ้นมา

## สิ่งที่ต้องทำ (Proposed Changes)

### 1. Database Schema
- **[MODIFY]** เพิ่มคอลัมน์ `Checklist` (NVARCHAR(MAX)) ในตาราง `TeamPlanner_Projects` เพื่อเก็บข้อมูล Checklist ในรูปแบบ JSON (เช่น `[{"id": 1, "text": "ออกแบบฐานข้อมูล", "isDone": false}]`)

### 2. Backend API
- **[MODIFY]** `api/projects.php`: อัปเดตให้รองรับการดึงและบันทึกข้อมูล `Checklist`

### 3. Frontend - Projects Tab
- **[MODIFY]** `src/components/ProjectsTab.jsx`:
  - เพิ่ม UI สำหรับจัดการ Checklist ในหน้าต่างสร้าง/แก้ไขโปรเจ็ค (เพิ่ม, ลบ, แก้ไขข้อความ)
  - เพิ่ม **Progress Bar** บนการ์ดโปรเจ็ค โดยคำนวณจาก `(จำนวน Checklist ที่ทำเสร็จ / จำนวน Checklist ทั้งหมด) * 100`
  - สามารถติ๊กถูก Checklist จากหน้า Projects Tab ได้โดยตรง

### 4. Frontend - Add Task Modal
- **[MODIFY]** `src/components/AddTaskModal.jsx`:
  - ย้ายตัวเลือก **"ผูกกับโปรเจ็ค"** ขึ้นไปเป็นลำดับแรกสุด
  - หากเลือกโปรเจ็ค:
    - เปลี่ยนชื่อ Title ของ Task ให้เป็น **ชื่อโปรเจ็ค** อัตโนมัติ (ผู้ใช้สามารถแก้ต่อได้ถ้าต้องการ)
    - แสดงรายการ Checklist ของโปรเจ็คนั้นที่ **"ยังไม่เสร็จ"** ขึ้นมาให้เลือก
    - เมื่อผู้ใช้เลือก Checklist ระบบจะนำข้อความของ Checklist นั้นไปใส่ใน Description หรือตั้งเป็น Subtask ของงานนี้ให้โดยอัตโนมัติ เพื่อให้ผู้ใช้รู้ว่า Task นี้ทำเพื่อตอบสนอง Checklist ข้อไหน

### 5. Frontend - Dashboard
- **[MODIFY]** `src/components/DashboardTab.jsx`:
  - เพิ่ม Widget สรุปจำนวนโปรเจ็ค (Active / Closed)
  - แสดงรายการ Active Projects พร้อม Progress Bar เพื่อให้เห็นภาพรวมของทีมในหน้าเดียว

## Open Questions (คำถามเพื่อความชัดเจน)
> [!IMPORTANT]
> **การอัปเดตความคืบหน้าของ Checklist:**
> เมื่อผู้ใช้สร้าง Task โดยอ้างอิงจาก Checklist ของโปรเจ็ค คุณอยากให้ระบบทำงานแบบไหนเมื่อ Task นั้นทำเสร็จ (ย้ายไปช่อง Done)?
> 
> **ตัวเลือกที่ 1 (อัตโนมัติ):** เมื่อ Task ถูกย้ายไปช่อง Done ให้ระบบไปติ๊กถูก Checklist ข้อนั้นในโปรเจ็คให้โดยอัตโนมัติ (วิธีนี้ล้ำ แต่อาจจะต้องเพิ่มฟิลด์ใน DB เพื่อเชื่อม Task เข้ากับ Checklist ID)
> **ตัวเลือกที่ 2 (แมนนวล - แนะนำ):** Task ก็คือ Task เมื่อทำเสร็จ ผู้ใช้แค่เข้าไปที่หน้า Projects Tab แล้วกดติ๊กถูกที่ Checklist ด้วยตัวเอง (วิธีนี้ยืดหยุ่นกว่า เพราะ 1 Checklist อาจจะประกอบด้วยหลาย Tasks ย่อยๆ ที่เปิดขึ้นมาทำในแต่ละวัน)
>
> รบกวนแจ้งให้ผมทราบหน่อยครับว่าชอบแบบไหน แล้วผมจะดำเนินการสร้างและปรับปรุงระบบให้ทันทีครับ!

## Verification Plan
- ตรวจสอบว่าคอลัมน์ Checklist ถูกเพิ่มใน SQL Server และ API ทำงานได้ถูกต้อง
- ทดสอบสร้างโปรเจ็คพร้อม Checklist และดู Progress Bar ว่าคำนวณถูกต้อง
- ทดสอบหน้า Add Task ว่าการเลือกโปรเจ็คดึง Checklist มาแสดงและเปลี่ยนชื่องานให้อัตโนมัติ
- ตรวจสอบ Dashboard ว่าดึงข้อมูลโปรเจ็คมาแสดงผลได้อย่างสวยงาม
