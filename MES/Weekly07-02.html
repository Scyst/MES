1. MES Team Planner & Navigation

Major Update
🎯 ปัญหา & เป้าหมายที่ต้องการแก้ไข

การวางแผนงานแบบรายวันทำให้ทีมมองไม่เห็นภาพรวมระยะยาว จัดการ Workload ได้ยาก นอกจากนี้ระบบ UI เดิมไม่รองรับโหมดกลางคืน (Dark Mode) ทำให้ปวดตาเวลาใช้งานกะดึก และการสลับหน้าจอระหว่างแอปต่างๆ ยังมีการโหลดหน้าเว็บที่ล่าช้า

✨ สิ่งที่พัฒนาสำเร็จในสัปดาห์นี้

  • พัฒนาหน้าจอ Weekly View และปรับปรุง UI ส่วน Workload Grid บน Dashboard ให้แสดงผลแบบ 2 คอลัมน์บน PC เพื่อความสมดุล
  • เพิ่มฟังก์ชัน MultiSelect Badge สำหรับการกำหนดผู้รับผิดชอบงาน และการเลือก Tag ทำให้ใช้งานง่ายขึ้น
  • แก้ไขปัญหาการดาวน์โหลดรูปภาพจาก Gantt Chart Export ให้แสดงผลสวยงามและสัดส่วนถูกต้องเหมือนบนหน้าเว็บจริง
  • อัปเดต URL ระบบนำทาง Mobile Entry ให้ชี้ตรงไปยังแอปพลิเคชัน MobileApp แบบ SPA เต็มรูปแบบ
  • ระบบสมบูรณ์ Light/Dark Mode โดยประยุกต์ใช้ View Transitions API เพื่อความลื่นไหลระดับ Native App
  • ยกระดับระบบแจ้งเตือนเป็น Messenger-style Chat Room แบบลอยตัวตลอดเวลา พร้อมระบบพรีวิวข้อความล่าสุด และแจ้งเตือนจุดสีแดง (Red Dot)
  • เพิ่มฟังก์ชัน Custom AKA ในกล่องแชท ให้ผู้ใช้ระบุชื่อย่อของตนเองเพื่อให้ระบบดึงงานที่เกี่ยวข้องมาแสดงได้อย่างแม่นยำ และปรับ Layout หน้าต่างแชทไม่ให้บังปุ่ม FAB
Business Impact: ลดเวลาในการวางแผนงานของหัวหน้าทีมด้วยหน้าจอ Weekly View ที่ตอบสนองทันที และยกระดับประสบการณ์ผู้ใช้ (UX) ฝั่งมือถือให้ทำงานได้ลื่นไหลเทียบเท่าแอปพลิเคชันจริง

2. Architecture & IIoT Migration (สถาปัตยกรรมข้อมูล)

Major Update
🎯 ปัญหา & เป้าหมายที่ต้องการแก้ไข

ข้อมูลยอดผลิตและของเสียที่ได้จากการให้พนักงานกรอกตัวเลขด้วยมือ มักจะมีความคลาดเคลื่อน ไม่ตรงกับรอบสโตรคของเครื่องจักรจริง (IIoT Strokes) และโครงสร้าง API ตัวเก่าที่อ้างอิงเอกสารด้วย uniqid ทำให้คำนวณ OEE ได้ยาก

✨ สิ่งที่พัฒนาสำเร็จในสัปดาห์นี้

  • จัดทำเอกสารแผนการย้ายระบบ Migration Plan เพื่อเปลี่ยนผ่านจากระบบเก่า สู่ระบบการบันทึกข้อมูล IIoT อัตโนมัติ
  • อัปเดต Data Structure Mapping เชื่อมโยงความสัมพันธ์ระหว่างตารางสต๊อก STOCK_TRANSACTIONS และข้อมูลจาก PE_IIOT_TELEMETRY
  • ปรับจูนสูตรคำนวณการผลิต OEE Quality Calculation และจับคู่ข้อมูล Transaction ตามรหัสเครื่องจักร (Machine ID)
  • รื้อถอนระบบ uniqid แบบเก่าใน API `sp_ExecuteProduction` เพื่อใช้รหัสเครื่องจักรจริงๆ ทำให้ข้อมูลแม่นยำขึ้น 100%
Business Impact: เป็นก้าวสำคัญในการเปลี่ยนโรงงานสู่ Smart Factory เต็มรูปแบบ โดยแก้ปัญหายอดผลิตคลาดเคลื่อน และวางรากฐานเพื่อดึงยอดผลิตจริงจากเซ็นเซอร์ (IIoT) อัตโนมัติแทนคน

3. PE Enterprise (ระบบซ่อมบำรุง)

Optimization
🎯 ปัญหา & เป้าหมายที่ต้องการแก้ไข

ข้อมูลเครื่องจักรบนระบบแผนที่ IIoT Map ยังเชื่อมโยงกับระบบแจ้งซ่อมได้ไม่สมบูรณ์ ช่างซ่อมบำรุงต้องเสียเวลาเข้าหลายเมนูเพื่อค้นหาว่าเครื่องจักรในใบแจ้งซ่อมนั้น ตั้งอยู่ตรงจุดไหนของแผนผังโรงงาน

✨ สิ่งที่พัฒนาสำเร็จในสัปดาห์นี้

  • อัปเกรดระบบแผนที่ IIoT Map Builder เพิ่มฟีเจอร์ Advanced Map Filtering และการค้นหาเครื่องจักร
  • สร้างจุดเชื่อมโยง (Link) ให้ผู้ใช้สามารถกดคลิกที่เครื่องจักรบนแผนที่ แล้วกระโดดเข้าหน้าใบแจ้งซ่อม (Work Orders) ได้ทันที
  • พัฒนาระบบ Import Item Master (Excel) สำหรับแผนกชิ้นส่วนอะไหล่ (Spare Parts) โดยใช้ Template เดียวกันกับหน้า Export แบบ 100% พร้อมระบบ Upsert (อัปเดตและสร้างใหม่) ในตัว
  • ปรับโครงสร้างตารางฐานข้อมูล `MT_ITEMS` เพิ่มคอลัมน์ `created_at` และ `last_updated` เพื่อติดตามประวัติการแก้ไขข้อมูลแบบเงียบๆ หลังบ้าน
  • อัปเกรดโครงสร้างซอร์สโค้ดของ PE ไปเป็น ES6 Modules (Javascript สมัยใหม่) และปรับจูนการเชื่อมต่อฐานข้อมูล SQL Server ให้เร็วขึ้น
  • แก้ปัญหาบั๊กหน้าจอ (Layout corruption) และหน้าต่าง Modal ซ้อนทับกัน (Backdrop stacking bug) ในหน้า IIoT Filter

Optimization

พนักงานคลังสินค้าต้องเสียเวลากดอนุมัติคำขอเบิกวัตถุดิบทดแทน (Scrap Replace) ทีละรายการ (Manual) ทำให้เกิดความล่าช้าสะสมในกระบวนการทำงานเมื่อมีคำขอหลั่งไหลเข้ามาพร้อมกันจำนวนมาก

✨ สิ่งที่พัฒนาสำเร็จในสัปดาห์นี้

  • ปรับปรุงลอจิกการรับของ (RM Receiving) แก้ปัญหาวัตถุดิบไปปรากฏในหมวดหมู่ SEMI หรือ WIP ผิดพลาด โดยการจัดลำดับความสำคัญ (Prioritize) ให้ประเภท FG, SEMI, WIP ไปอยู่ลำดับท้ายสุดแทน
  • ปรับแก้การทำงานของระบบ Auto-Logout ใน appCore.js ให้ใช้การอ้างอิง Path แบบ Dynamic (URL Resolution) ช่วยลดปัญหาการเด้งหลุดหน้าขาว (401 Unauthorized) ในกรณีเรียกใช้งานข้ามโฟลเดอร์

6. SAP ERP Live Data Integration (เชื่อมโยงฐานข้อมูล SAP)

Major Update
🎯 ปัญหา & เป้าหมายที่ต้องการแก้ไข

ระบบ MES ขาดการมองเห็นข้อมูลฝั่ง ERP ทำให้พนักงานต้องสลับหน้าจอหรือใช้โปรแกรมแยกเพื่อเช็คใบสั่งผลิต (Operation Slips) และยอดสต๊อกวัตถุดิบ (Stock) ส่งผลให้เกิดความล่าช้าและต้องพึ่งพาทีม IT ตลอดเวลา

✨ สิ่งที่พัฒนาสำเร็จในสัปดาห์นี้

  • MCP System Setup: ตั้งค่า sap-mssql ผ่านระบบ AI Agent MCP Server เพื่อให้สามารถรันคำสั่ง Query ไปยัง SAP ได้อัตโนมัติในอนาคต
  • Independent Connection: สร้างไฟล์เชื่อมต่อ PDO (sap_db.php) ยิงตรงเข้า Server `10.0.0.4` (SNC-SAP) โดยไม่หน่วงฐานข้อมูลหลักของ MES
  • Smart Web Interface: พัฒนา Dashboard SAP_Sync ด้วยดีไซน์ล้ำสมัย (Dark Mode) รองรับการสลับ Tab พร้อมดึงชื่อคอลัมน์ของ SAP ออกมาแสดงผลแบบ Dynamic 100%
  • Architectural Documentation: สรุปเอกสารโครงสร้างตาราง View_OperationSlip_1820 (35 คอลัมน์) และ View_SAP_ALL_STOCK (8 คอลัมน์) ลงใน SAP_INTEGRATION_DOC.md อย่างสมบูรณ์แบบ
Business Impact: ลดรอยต่อระหว่างหน้างานฝ่ายผลิต (MES) และระบบบัญชีส่วนกลาง (ERP) ทำให้พนักงานหน้างานสามารถตรวจสอบยอดสต๊อกและคำสั่งผลิตแบบเรียลไทม์ได้ด้วยตนเอง

พบปัญหาผู้ใช้งานไม่สามารถกดขยายเวลาลิงก์ให้ลูกค้า (Extend CAR Link) ได้ ระบบแจ้งเตือน 500 Server Error เนื่องจากมีการส่งค่าประเภท String (nvarchar) เข้าไปยังฟังก์ชัน DATEADD ของ SQL Server ที่รองรับเฉพาะ Integer

✨ สิ่งที่พัฒนาสำเร็จในสัปดาห์นี้

  • SQL Type Casting: แก้ไขคำสั่ง SQL ใน qms_action.php โดยเพิ่มการทำ Type Casting CAST(? AS INT) เพื่อแปลงพารามิเตอร์ให้อยู่ในรูปแบบ Integer ก่อนส่งให้ DATEADD ทำให้ระบบขยายเวลากลับมาใช้งานได้ตามปกติ

8. MES Learning Hub & Team Planner (Standalone)

Major Update
🎯 ปัญหา & เป้าหมายที่ต้องการแก้ไข

ระบบศูนย์การเรียนรู้ยังขาดเนื้อหาเชิงสถาปัตยกรรม และตัวแอปพลิเคชัน Planner ไม่ควรนำมารวมในแอปเดียวกับระบบเรียนรู้ (Learning Hub) เพราะมีเป้าหมายการใช้งานที่แตกต่างกัน

✨ สิ่งที่พัฒนาสำเร็จในสัปดาห์นี้

  • Learning Hub (mes-learning-hub): เพิ่มเนื้อหา Course 3 (React + Vite Architecture) และ Course 4 (Enterprise System & Security) พร้อมย้ายหน้าคู่มือ Team Workflow เข้ามาไว้ในบทเรียนที่ 4 ของคอร์ส 3 อย่างสมบูรณ์
  • Standalone Planner (mes-team-planner): สร้างโปรเจกต์ SPA (Single Page Application) แยกร่างออกมาเด็ดขาด พัฒนาโครงสร้าง Layout Sidebar, Calendar View และ Kanban Task Board พร้อม Mock Data สำหรับใช้งาน
  • Vite Production Build: ปรับปรุงการตั้งค่า base: './' ใน vite.config.js ทำให้ Build และนำขึ้น Server ได้โดยไร้ปัญหาจอขาว
Business Impact: ขยายความสามารถระบบ E-Learning ให้ครอบคลุมความรู้ด้านโครงสร้างระบบ และจัดระเบียบแอปพลิเคชันช่วยงานของทีมแบบ Standalone เพื่อการดูแลรักษาโค้ดที่ง่ายขึ้น (Modularity)
